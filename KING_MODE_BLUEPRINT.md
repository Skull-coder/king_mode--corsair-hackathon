# Project Blueprint: 👑 KING MODE

**AI-Powered Email & Calendar Operating System**
**Hackathon Target:** Corsair Hackathon
**Objective:** Build a lightning-fast, high-end productivity platform that transforms manual email and calendar management into an autonomous, context-aware AI assistant ecosystem.

---

## 1. Core Architecture & Workflow

KING MODE operates via a dual-interface architecture designed around user autonomy and high-speed execution:

### Workspace Mode (Traditional UI)

- A high-performance, unified dashboard showing today's meetings, priority items, and deep insights.
- Zero-refresh reactive updates via Server-Sent Events (SSE).
- Manual power-user interaction via custom global keyboard shortcuts and a centralized command palette (`Cmd+K`).

### KING MODE (AI Control Center)

- A cinematic interface transition that shifts the platform into a terminal-style AI executive assistant command center.
- Accepts natural language text and fluid voice commands.
- Implements multi-step, cross-app tool calling (e.g., parsing context from emails, drafting a response, scheduling a corresponding meeting, and sending invites autonomously through a single command).

```
[Realtime Data Ingestion Flow]
Corsair Webhooks ──> Next.js Route Handler ──> Drizzle ORM (Postgres + pgvector)
                                                      │
                                           ┌──────────┴──────────┐
                                           ▼                     ▼
                                  Local Semantic Cache      SSE Broadcast
                                  (Vector Embeddings)       (Instant UI Update)
```

## 2. Technology Stack

### Frontend

- **Next.js 15 (App Router) & TypeScript:** Core framework providing strict type safety, layout optimization, and server/client split rendering.
- **Tailwind CSS & shadcn/ui:** Primitives optimized for desktop-grade accessibility and custom utility styling.
- **Framer Motion:** Handles layout state animations and morphing transitions during the KING MODE activation.
- **TanStack Query (`@tanstack/react-query`):** Manages local asynchronous state, data caching, background fetching, and optimistic client-side updates.
- **`cmdk` & `react-hotkeys-hook`:** Powers keyboard navigation, row-hopping shortcuts (`J`/`K`), archiving (`E`), and the global execution palette.
- **`date-fns` & `react-day-picker`:** Lightweight date manipulation and interactive calendar rendering.

### Backend & Database

- **Next.js Route Handlers:** Handles incoming server requests, lightweight processing, and asynchronous background actions.
- **Next.js Server Actions:** For end-to-end type-safe client mutations bypassing heavy REST endpoints.
- **PostgreSQL + `pgvector`:** Relational database storage combined with native vector extension layers to handle multi-modal embedding datasets.
- **Drizzle ORM:** Fast TypeScript object-relational mapping ensuring minimal compile overhead and maximum query raw speed.

### Integrations & AI

- **Corsair API / MCP:** Out-of-the-box management layer bridging the application to the underlying Google APIs (Gmail + Google Calendar), handling background synchronization, semantic searches, and agent tool execution.
- **Clerk:** End-to-end user session provisioning, authentication gates, and OAuth authorization routines.
- **Gemini / OpenAI:** Large language models orchestrating email classification hooks, structural thread summarization, meeting brief generation, and autonomous tool calling agents.

## 4. Key Architectural Decisions

### Why no tRPC?

tRPC adds significant boilerplate for monorepo setup. Under a 10-day hackathon deadline, Next.js Server Actions provide the same end-to-end type safety for mutations with zero configuration overhead. Route Handlers cover the remaining cases (webhooks, SSE, public API endpoints).

### Why `lib/db/queries/` as a separate layer?

Route handlers that contain raw Drizzle queries become duplicated and untestable fast. Centralizing all DB access in `queries/` means route handlers only do transport + orchestration — they call a function and return the result. If a query needs to change, it changes in one place.

### Why `lib/corsair/webhooks.ts` is separate?

Signature verification logic is security-critical and must not live inline in a route handler. Isolating it makes it easy to test, easy to rotate secrets, and keeps the route handler focused on orchestration.

### Why `config/shortcuts.ts`?

Keyboard shortcuts are a bonus judging criterion. Having all shortcut definitions in one file means you can show judges a complete list instantly. It also makes `use-hotkeys.ts` a thin binding layer — no magic strings scattered across components.

### Why `loading.tsx` per route?

Next.js App Router automatically renders `loading.tsx` as a Suspense boundary while the page fetches data. This gives you skeleton screens with zero extra code in your page components — judges see polished loading states instead of blank white flashes.

## 5. Realtime Architecture (4-Phase Pipeline)

```
Phase 1 — Inbound Event
Corsair detects Gmail/Calendar change
→ POST /api/webhooks/corsair
→ lib/corsair/webhooks.ts verifies signature
→ Payload parsed into typed CorsairEmailEvent or CorsairCalendarEvent

Phase 2 — Synchronous Replication
lib/db/queries/emails.ts → insertEmail()  (or calendar equivalent)
Priority field set to "pending" until classification completes

Phase 3 — Async Background Processing (non-blocking)
lib/ai/embeddings.ts → generateEmbedding(subject + body snippet)
lib/db/queries/embeddings.ts → insertEmbedding()
lib/ai/classify.ts → classifyEmailPriority()
lib/db/queries/emails.ts → updateEmailPriority()

Phase 4 — Client Notification
lib/sse/emitter.ts → emit("email:new", { id, priority })
/api/events SSE route → broadcasts to all connected browser clients
components/shared/realtime-provider.tsx → queryClient.invalidateQueries(["emails"])
TanStack Query → background refetch → UI updates instantly, no page reload
```

                          │
              ┌───────────▼───────────┐
              │   Send to Corsair MCP  │

                   Stream response to
                   components/king/agent-chat.tsx
